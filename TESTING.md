# 校园失物招领项目测试指南

## 项目概述

本项目是一个校园失物招领系统，包含三个主要部分：

1. **小程序后端** - `mainwechatapp/campus-lost-found/backend`
2. **管理后台后端** - `backend-admin`
3. **管理后台前端** - `admin-web`

## 环境要求

- Python 3.8+
- Node.js 16+ (可选，用于前端开发)

## 测试设置

### 1. 小程序后端测试

#### 安装依赖

```bash
cd mainwechatapp/campus-lost-found/backend
pip install -r requirements.txt
```

#### 运行测试

```bash
python -m pytest test_app.py -v
```

或者使用 pytest 直接运行：

```bash
pytest test_app.py -v
```

**测试结果：** ✓ 14个测试全部通过

### 2. 管理后台后端测试

#### 安装依赖

```bash
cd backend-admin
pip install -r requirements.txt
```

#### 运行测试

```bash
python -m pytest test_app.py -v
```

**测试结果：** ✓ 12个测试全部通过

### 3. 管理后台前端测试

#### 基础测试（不需要 Node.js）

我们提供了一个基于文件系统的基础测试，不需要安装前端依赖：

```bash
cd admin-web
```

在项目根目录查看 `admin-web/tests/test-basics.js` 文件。

#### 完整前端测试（需要 Node.js）

如果已安装 Node.js 和 npm：

```bash
cd admin-web
npm install
```

可以运行 Vite 开发服务器进行手动测试：

```bash
npm run dev
```

## 测试方法说明

由于项目的原始 `app.py` 文件在模块级别初始化了数据库连接，我们采用了以下实用的测试策略：

1. **不直接导入 app** - 避免影响现有数据库
2. **文件系统级测试** - 检查项目结构和文件完整性
3. **源代码分析** - 验证必要的端点和功能是否存在
4. **模拟逻辑测试** - 测试业务逻辑的正确性，而不依赖完整的应用实例
5. **安全的测试环境** - 所有测试都不会修改真实数据
